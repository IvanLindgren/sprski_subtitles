# Читавук-речник

Читавук-речник — веб-сервис для создания сербских субтитров, синхронного просмотра видео и ведения личного словаря. Пользователь загружает видео, получает текст с таймкодами, сохраняет незнакомые слова с переводами и заметками, а затем скачивает VTT, SRT или готовый MP4 со встроенными субтитрами.

## Локальный запуск

Для запуска нужен Node.js. Выполните `npm install`, затем скопируйте `.env.example` в `.env` и при желании добавьте ключ из [Groq Console](https://console.groq.com/keys). После команды `npm run dev` сайт откроется по адресу [http://127.0.0.1:5173](http://127.0.0.1:5173). Ключ также можно ввести непосредственно в интерфейсе. Он сохраняется в LocalStorage этого браузера, не удаляется после закрытия вкладки и остаётся на устройстве до явного удаления в настройках сайта.

## Возможности

Сервис принимает MP4, MOV, WEBM, MKV и другие форматы, извлекает речевую дорожку и отправляет её в `whisper-large-v3` с языком `sr`. Аудиодорожка сжимается в Opus с битрейтом, подобранным под длительность видео (чем короче ролик, тем выше качество звука в пределах лимита Groq на размер файла). Временная шкала аудио предварительно обнуляется, а после основного распознавания сервер ищет в результате пропуски длиннее 12 секунд — будь то самое начало видео или тишина посреди ролика — и повторно распознаёт только эти фрагменты отдельными короткими запросами, что заметно снижает вероятность того, что субтитры «съедят» реальную речь. Видео с YouTube можно не скачивать вручную: по ссылке `youtube.com`/`youtu.be` сервер сам скачивает ролик через `yt-dlp` (с приоритетом H.264/AAC для совместимости плеера), передаёт его в браузер и сразу запускает распознавание. Распознавание выполняется как фоновая задача: браузер показывает реальный прогресс загрузки и FFmpeg, приблизительное оставшееся время ответа Groq, а Render Logs фиксирует этап, процент, прошедшее время и оценку остатка. Проекты сохраняются в LocalStorage, исходные видео размещаются в IndexedDB браузера, а системная установка FFmpeg не требуется благодаря `ffmpeg-static`. После распознавания пользователь может анонимно опубликовать видео с категорией «фильм», «мультфильм», «блог», «интервью», «новости», «обучение» или «другое». Публичная библиотека показывает видео вместе с сохранёнными синхронными субтитрами и не передаёт ключ Groq или личный словарь.

## Публичное хранилище Cloudflare R2

Для постоянной публичной библиотеки используется S3-совместимое хранилище Cloudflare R2, потому что файловая система бесплатного Render очищается при перезапуске. Создайте публичный bucket, выпустите токен R2 с правом чтения и записи объектов и добавьте в Render значения `S3_ENDPOINT`, `S3_BUCKET`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY` и `PUBLIC_MEDIA_BASE_URL`. Для `S3_ENDPOINT` используется адрес вида `https://ACCOUNT_ID.r2.cloudflarestorage.com`, для `S3_REGION` оставьте `auto`, а в `PUBLIC_MEDIA_BASE_URL` укажите публичный адрес bucket через собственный домен или временный домен `r2.dev`. После сохранения переменных Render автоматически перезапустит сервис, а поле `publicLibrary` в ответе `/api/health` изменится на `true`.

Бесплатный объём R2 включает 10 ГБ хранения в месяц, один миллион операций записи класса A, десять миллионов операций чтения класса B и бесплатный исходящий трафик. Домен `r2.dev` предназначен для разработки и имеет ограничения частоты запросов, поэтому для открытого рабочего сайта лучше подключить к bucket собственный домен. Анонимная публикация требует подтверждения права на размещение и ограничена тремя видео в час с одного IP-адреса.

## Бесплатный тариф

На июль 2026 года Groq публикует для Whisper лимит в 2 000 запросов в сутки, 7 200 секунд аудио в час и 28 800 секунд в сутки. Прямое вложение ограничено 25 МБ, поэтому сервер предварительно преобразует дорожку в моно-аудио Opus с частотой 16 кГц. Актуальные значения находятся в документации [Groq rate limits](https://console.groq.com/docs/rate-limits) и [Speech to Text](https://console.groq.com/docs/speech-to-text).

## Деплой с Netlify и Render

Netlify используется для React-интерфейса, а обработка видео запускается в Render Web Service. Репозиторий содержит `netlify.toml` и `render.yaml`, поэтому обе части собираются автоматически. Сначала разверните Render Blueprint через [Render Dashboard](https://dashboard.render.com/blueprints) и скопируйте выданный адрес вида `https://имя.onrender.com`. Затем добавьте в Netlify переменную окружения `VITE_API_BASE_URL` с этим адресом и запустите новый deploy. После этого запросы из `sprskisubtitles.netlify.app` будут отправляться напрямую в Express-сервер Render. В Render можно указать общий `GROQ_API_KEY`, но это необязательно, поскольку каждый пользователь может вводить собственный ключ в интерфейсе. Бесплатный Render Web Service засыпает после пятнадцати минут без запросов и запускается снова при следующем обращении.
