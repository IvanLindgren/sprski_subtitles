# Читавук-речник

Читавук-речник — веб-сервис для создания сербских субтитров, синхронного просмотра видео и ведения личного словаря. Пользователь загружает видео на сербском, английском, русском или другом языке, получает сербский текст с таймкодами, сохраняет незнакомые слова с переводами и заметками, а затем скачивает VTT, SRT или готовый MP4 со встроенными субтитрами.

## Локальный запуск

Для запуска нужен Node.js. Выполните `npm install`, затем скопируйте `.env.example` в `.env` и укажите общий `GROQ_API_KEY` из [Groq Console](https://console.groq.com/keys). После команды `npm run dev` сайт откроется по адресу [http://127.0.0.1:5173](http://127.0.0.1:5173). Посетителю не нужно вводить ключ: серверный ключ применяется автоматически. При желании пользователь может указать в интерфейсе собственный ключ, который имеет приоритет над общим, сохраняется только в LocalStorage его браузера и остаётся там до явного удаления.

## Возможности

Сервис принимает MP4, MOV, WEBM, MKV и другие форматы, извлекает речевую дорожку и отправляет её в `whisper-large-v3` без принудительного языка, поэтому Groq автоматически определяет исходную речь. После распознавания сервер переводит каждый фрагмент на сербскую латиницу через `openai/gpt-oss-120b`, сохраняя исходные таймкоды; сербская речь остаётся без повторного перевода. Аудиодорожка сжимается в Opus с битрейтом, подобранным под длительность видео. Временная шкала аудио предварительно обнуляется, а после основного распознавания сервер ищет в результате пропуски длиннее 12 секунд и повторно распознаёт только эти фрагменты. Видео с YouTube можно не скачивать вручную: по ссылке `youtube.com`/`youtu.be` сервер сам скачивает ролик через `yt-dlp`, передаёт его в браузер и сразу запускает распознавание. Распознавание и перевод выполняются как фоновая задача: браузер показывает прогресс, а Render Logs фиксирует этап, процент, прошедшее время и оценку остатка. Проекты сохраняются в LocalStorage, исходные видео размещаются в IndexedDB браузера, а системная установка FFmpeg не требуется благодаря `ffmpeg-static`. После распознавания пользователь может анонимно опубликовать видео с готовыми сербскими субтитрами. Публичная библиотека не передаёт ключ Groq или личный словарь.

## Публичное хранилище Cloudflare R2

Для постоянной публичной библиотеки используется S3-совместимое хранилище Cloudflare R2, потому что файловая система бесплатного Render очищается при перезапуске. Создайте публичный bucket, выпустите токен R2 с правом чтения и записи объектов и добавьте в Render значения `S3_ENDPOINT`, `S3_BUCKET`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY` и `PUBLIC_MEDIA_BASE_URL`. Для `S3_ENDPOINT` используется адрес вида `https://ACCOUNT_ID.r2.cloudflarestorage.com`, для `S3_REGION` оставьте `auto`, а в `PUBLIC_MEDIA_BASE_URL` укажите публичный адрес bucket через собственный домен или временный домен `r2.dev`. После сохранения переменных Render автоматически перезапустит сервис, а поле `publicLibrary` в ответе `/api/health` изменится на `true`.

Бесплатный объём R2 включает 10 ГБ хранения в месяц, один миллион операций записи класса A, десять миллионов операций чтения класса B и бесплатный исходящий трафик. Домен `r2.dev` предназначен для разработки и имеет ограничения частоты запросов, поэтому для открытого рабочего сайта лучше подключить к bucket собственный домен. Анонимная публикация требует подтверждения права на размещение и ограничена тремя видео в час с одного IP-адреса.

## Бесплатный тариф

На июль 2026 года Groq публикует для Whisper лимит в 2 000 запросов в сутки, 7 200 секунд аудио в час и 28 800 секунд в сутки. Прямое вложение ограничено 25 МБ, поэтому сервер предварительно преобразует дорожку в моно-аудио Opus с частотой 16 кГц. Актуальные значения находятся в документации [Groq rate limits](https://console.groq.com/docs/rate-limits) и [Speech to Text](https://console.groq.com/docs/speech-to-text).

## Деплой на Render

React-интерфейс и Express API разворачиваются вместе в одном Render Web Service: сервер раздаёт собранный каталог `dist` и обрабатывает запросы `/api` с того же адреса. Репозиторий содержит `render.yaml`, поэтому достаточно создать Render Blueprint, заполнить секретные переменные, включая общий `GROQ_API_KEY`, и открыть выданный адрес вида `https://имя.onrender.com`. Netlify для работы сайта не требуется. Бесплатный Render Web Service засыпает после пятнадцати минут без запросов и запускается снова при следующем обращении.
